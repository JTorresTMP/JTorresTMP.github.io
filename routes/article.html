Let, Const, and Var

By Jose Torres

Introduction

In JavaScript there are three main ways to declare a variable, and until ES6 in 2015 the keyword var was the only way to declare variables. This article aims to help you understand the differences and when you would use one keyword over the other.

The 'var' Keyword

The var keyword is somewhat of a relic of the past. It is very similar to the let keyword, with a few minor exceptions. As you may already know, variables are confined to the scope in which they are declared in. For instance, in this example code: 

                        
                            var myVar = 'John';

                            function logVar () {
                                console.log(myVar)
                            }

                            logVar() //'John'
The variable var is declared in the global scope, thus it is able to be called anywhere even inside the function. Another important thing is that var is globally scoped by default unless it is declared in a function, in which case it is function scoped.

The 'let' Keyword

The let keyword is similar to the var keyword in almost every way, except that it is block-scoped. Note the difference in wording. A block of code is any code encased by curly braces {}. So any variable declared using the let keyword will only be visible inside its respective block. This is why for loops commonly use let when they are used. 

                    
                        for (let i = 0; i < 10; i++) {
                            ... 
                        }

The 'const' Keyword

Const stands for constant. This keyword follows the same scope rules as let, that is, it is block-scoped. The main difference is that the value assigned at declaration to a const variable cannot be changed. If you try to assign another value, JavaScript will throw an error. 

                        
                            const name = 'Joseph';

                            name = 'Matt' //This line will give you an error

When to Use One Over the Other?

For the most part, using let anywhere you would use var will be better. You will run into less errors over time by using let. Use const if you do not want the value of your variable to change accidentally. One common use of const is to define arrow functions like so: 

                        
                            const whateverFunction = () => {
                                //do something
                            }
If the above code confuses you, please head over to our article on arrow functions. We hope you understand now the differences between the three keywords used to declare a variable and the reasons when you would use one over the other.
